# What is Programming?

## Theory

**Programming** is the act of writing **precise instructions** for a computer so it can perform a task automatically. Those instructions must be written in a **programming language**—a formal notation humans and tools can understand, which is then translated or interpreted into operations the machine can execute.

Unlike a casual conversation, code must be **unambiguous**: the computer does exactly what the instructions say, not what you *meant* if you were vague.

At a high level, programs usually:

1. **Receive** data (from the keyboard, a file, a network, sensors, …).
2. **Process** it (calculate, compare, transform, decide).
3. **Produce** a result (text on screen, a file, a message, movement on screen, …).

You do not need to know hardware details on day one. What matters first is learning to **think in steps** and express those steps clearly.

## Beginner-friendly notes

- **Program** = the full set of instructions (often many files). **Code** = the text of those instructions.
- **Bug** = behavior you did not intend, usually because the instructions were wrong or incomplete.
- **Algorithm** = the idea of *how* to solve a problem as clear steps (you will connect this to Python soon).
- Feeling stuck is normal; programming is a skill you build with **practice**, like music or sport.

## Example (conceptual, then Python)

**Task in plain language:** Say hello using the name `"Alex"`.

In Python (preview—you will run this after installation):

```python
name = "Alex"
print("Hello,", name)
```

## Expected output

```text
Hello, Alex
```
